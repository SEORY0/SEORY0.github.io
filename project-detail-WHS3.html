윈도우 커널 1-day 취약점 분석 및 익스플로잇

CVE-2024-35250(KernelStreaming) 취약점 분석 및 권한 상승(LPE) 익스플로잇 개발 프로젝트

기간

2025.06. - 2025.08.

역할

취약점 분석 및 익스플로잇 개발

기술 스택

Windows Kernel C/C++ IDA Pro WinDbg

프로젝트 배경

화이트햇스쿨 3기 과정의 3개월 프로젝트로, 실제 버그 바운티 시나리오에 대비하기 위해 1-day 취약점을 선정하여 분석하고 익스플로잇 코드를 개발하는 것을 목표로 했습니다. 팀원들과 협력하여 취약점의 근본 원인을 파악하고, 이를 통해 시스템 권한을 상승시키는 전체 과정을 성공적으로 구현하며 시스템 보안에 대한 깊이 있는 이해를 얻고자 했습니다.


주요 과정 및 기술적 도전

프로젝트는 취약점 선정, 환경 구축, 패치 비교(Patch Diffing), 루트커즈 분석(RCA), 익스플로잇 개발의 5단계로 진행되었습니다. 제가 핵심적으로 담당했던 부분은 루트커즈 분석과 익스플로잇 개발이었습니다.

1. 루트커즈 분석 (Root Cause Analysis)

선정한 CVE-2024-35250 취약점은 Windows Kernel Streaming 드라이버(ks.sys)에 존재하는 Use-After-Free(UAF) 버그였습니다. 패치 비교를 통해 취약점이 존재하는 코드 영역을 특정했고, WinDbg를 이용해 커널 디버깅 환경에서 해당 코드의 실행 흐름을 추적했습니다. 이 과정에서 특정 IOCTL 요청을 보냈을 때 객체가 해제된 후에도 참조가 남아있는 것을 확인했으며, 이것이 취약점의 근본 원인임을 밝혀냈습니다.

2. 익스플로잇 개발

분석된 취약점을 트리거하여 시스템 권한을 상승시키는 PoC(Proof of Concept) 코드를 C++로 작성했습니다. UAF를 이용해 해제된 메모리 영역을 제어 가능한 데이터로 덮어쓰고(Heap Spraying), 조작된 객체를 통해 임의의 코드를 실행(Arbitrary Code Execution)하는 전략을 사용했습니다. 커널 메모리 보호 기법(KASLR)을 우회하고 안정적인 익스플로잇을 만드는 과정이 가장 큰 기술적 도전이었습니다.


성과 및 배운 점

최종적으로 CVE-2024-35250 취약점을 성공적으로 재현하고, 로컬 권한 상승(LPE)을 통해 관리자(SYSTEM) 권한을 획득하는 풀 익스플로잇 코드를 완성했습니다.

  • Windows 커널 내부 동작 방식과 메모리 관리 기법에 대한 실질적인 이해를 높일 수 있었습니다.
  • IDA Pro, WinDbg와 같은 리버싱 및 디버깅 도구 사용에 대한 숙련도를 향상시켰습니다.
  • 팀원들과의 협업을 통해 복잡한 기술적 문제를 해결하는 문제 해결 능력과 소통 능력을 길렀습니다.
